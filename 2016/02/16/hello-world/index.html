关于博客网站的搭建

2016-02-16

其实早在去年，也就是15年2月份的时候就已经开始折腾博客了，从刚一开始用github上推荐的jekyll折腾到现在用的hexo，真的是踩了不少坑。只是当时开学后忙于准备考研，搭建好的博客也被弃之脑后。如今考研结束，尽管努力了一年，也并没有得到一个好的结果。后悔是有，但更多应该是对时间的惋惜，如果这一年的时间都用来提升自己的实践技术，可能现在找工作也不会心里打鼓了吧。可是没有人能预知未来，所以也就无所谓后悔，无所谓惋惜，经历亦是一种收获。
由于隔了整整一年的时间，各种配置都忘的差不多了，以前申请的域名travistom.me将近到期可续费让我这个穷学生感到略贵，原先克隆的主题太卡，hexo又从2.x升到3.x……当然了，最主要的还是归于自己本身爱折腾的本质。遂把原先的仓库删掉，另申请了域名，重新来了一遍。网上的关于这方面的教程足够多，也足够细致。所以我打算写下流程并配以相关步骤的链接，以及自己记录下来的踩得那些坑（没记录下来的坑更多），以供其他爱好者参考吧。

猜测是国外的服务器不稳定，如果不太着急的话可以先不管它，第二天或者去吃个饭回来就好了。
当然也从网上找到了其他的解决方案，你可以试一下
通过执行telnet github.com 443，发现网络不通，于是想到设置代理，通过baidu找到设置proxy方法：

a.打开文件.gitconfig，文件位于用户主目录下，执行ls ~即可看到它。假设Win7登录用户名为tom，则位于C:\Users\tom目录下

b.编辑.gitconfig，在文件尾加入以下内容：
[http]
proxy = http://www.proxy.com:8080/
[http]顶格写，proxy前保留一个tab，其中www.proxy.com为代理服务器的IP或域名。完成后，再次执行git clone https://github.com/eyjian/mooon.git,发现成功了。。
对文件.gitconfig的操作，实际可以借助git命令来完成，而不必直接编辑.gitconfig，对应的子命令是git config –global，如：git config –global user.name “eyjian”，则在.gitconfig文件中可以看到如下内容：
[user]
name = eyjian
同理，配置proxy命令为：git config –global http.proxy http://www.proxy.com:8080/
